WebDAV and its uses

What is WebDAV?

Web-based Distributed Authoring and Versioning, or WebDAV is a set of extensions to HTTP protocol that allow for editing and managing of files collaboratively on a remote server.

History of WebDAV

WebDAV began as a meeting between UC Irvine graduate Jim Whitehead and interested members of the World Wide Web Consortium. The meeting was called to discuss the reading and writing capabilities in Tim Berners-Lee's original WorldWideWeb browser and how to implement those capabilities in the modern era.

The meetings between Whitehead and the members for the W3C resulted in the formation of an IETF working group, and eventually the protocol was first defined in Request for Comments 2518, commonly known as RFC 2518, however the protocol did not initially provide for versioning.

Eventually, in 2017, the WebDAV working group concluded its work and WebDAV was finally updated to RFC 4918.

What are the advantages of using WebDAV?

As WebDAV works over HTTP, it has some significant benefits over FTP, including:
  • Encryption
  • Strong Authentication
  • Proxy Support
  • Caching
  • Pipelining Multiple Transfers through a single TCP connection

What are the disadvantages of using WebDAV?

As WebDAV is collaborative, it has some potential disadvantages compared to FTP, including:
  • Security issues with the need to protect content through both file system and web server
  • Potential duplicate user management and authentication
  • No source control - files are only locked when they are open
  • Caching
  • Pipelining Multiple Transfers through a single TCP connection

What are some servers that support WebDAV?

There are a number of various servers that support WebDAV:
  • Apache
  • Microsoft's Internet Information Systems
  • SabreDAV
  • Nginx
  • ownCloud
  • Nextcloud
